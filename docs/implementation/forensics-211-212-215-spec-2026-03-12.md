## Issue #215 — Run Comparison (Wave 1-A)

### New packages / files
- `internal/forensics/rollout/` — shared rollout loader
  - `loader.go` — parse JSONL → `[]RolloutEvent`
  - `canonicalizer.go` — strip timestamps/run_ids for diff comparison
  - `loader_test.go`, `canonicalizer_test.go`
- `internal/forensics/differ/` — diff engine
  - `differ.go` — `Diff(a, b []RolloutEvent) DiffResult`
  - `scorer.go` — `Score(diff DiffResult) RegressionScore`
  - `differ_test.go`, `scorer_test.go`
- `cmd/forensics/` — CLI
  - `main.go` — subcommand: `diff <rollout_a> <rollout_b>`
  - `main_test.go`

### Key types
```go
type RolloutEvent struct {
    ID        string         `json:"id"`
    Type      string         `json:"type"`
    Step      int            `json:"step,omitempty"`
    Payload   map[string]any `json:"payload,omitempty"`
    Timestamp time.Time      `json:"timestamp"`
}

type DiffResult struct {
    StepDiffs    []StepDiff
    CostDelta    float64
    OutcomeDiff  string  // "identical" | "diverged" | "a_failed" | "b_failed"
    ScoreSummary RegressionScore
}

type RegressionScore struct {
    Winner  string  // "a" | "b" | "tie"
    Reasons []string
}
```

### Canonicalization rules
- Strip: `id`, `timestamp`, `run_id` from payloads
- Keep: `type`, `step`, `tool`, `args`, `result`, `cost`
- Normalize: event ordering by step+sequence

## Issue #211 — Causal Event Graph (Wave 1-B + Wave 2-D)

### New package
- `internal/forensics/causalgraph/`
  - `graph.go` — `CausalGraph`, `Node`, `Edge`, `Dependency` types
  - `builder.go` — `Builder.Record(step, event)` + `Build() CausalGraph`
  - `dataflow.go` — Tier 2: output→input string matching heuristics
  - `blame.go` — Tier 3: `BlameChain(targetEventID string) []Node`
  - `*_test.go` for each file

### New event types
- `EventCausalGraphSnapshot = "causal.graph.snapshot"` — emitted at run end if `CausalGraphEnabled`

### Runner config additions
```go
CausalGraphEnabled bool
```

### Tier 1: Mechanical dependencies
- After each LLM turn, record `{step, tool_call_id, context_message_ids}`
- Context message IDs come from the message slice passed to the provider
- Links: "tool call at step N depended on messages M1..Mk"

### Tier 2: Data flow heuristics
- After each tool result, check if any token (≥6 chars, not a common word) from the result appears in subsequent tool call args
- Track `DataFlowEdge{FromStep, FromCallID, ToStep, ToCallID, MatchedToken}`

### Tier 3: Blame path
- Reverse BFS from a target event ID back to origin
- Export as JSON adjacency list: `[]BlameNode{ID, Type, Step, Cause}`

## Issue #212 — Run Replay/Fork (Wave 2-C)

### Reuses
- `internal/forensics/rollout/` loader from Wave 1

### New package
- `internal/forensics/replay/`
  - `replayer.go` — `Replayer{runner *Runner}`, `Replay(rollout []RolloutEvent) error` (simulation)
  - `forker.go` — `Fork(rollout []RolloutEvent, fromStep int) (*RunContext, error)`
  - `replayer_test.go`, `forker_test.go`

### Phase 2: Offline simulation
- Load JSONL → reconstruct message history
- For tool calls: return recorded outputs (no live execution)
- Verify event sequence matches original

### Phase 3: Fork from step N
- Load rollout up to step N
- Reconstruct `[]harness.Message` history
- Hand off to live runner starting from that state
- Similar to `ContinueRun` but from arbitrary step

### Phase 4: HTTP endpoint (Wave 3)
- `POST /api/v1/runs/replay` — body: `{rollout_path, mode: "simulate"|"fork", fork_step}`
- Returns SSE stream of replay events
