2. PROPOSED BEHAVIORAL SPEC FORMAT
   - Preconditions (what must be true before calling)
   - Side effects (what state changes result)
   - Error modes (failure codes and recovery)
   - Atomicity & consistency guarantees
   - Performance characteristics (time, tokens, cost, limits)
   - Interaction patterns (when to call with other tools)
   - Agent decision guidance (when/why to choose this tool)

   - edit.md         (line-hash semantics, multiline matching)
   - apply_patch.md  (atomicity, rollback semantics)
   - glob.md         (recursion depth, pattern scoping)

10. KEY RISKS & MITIGATIONS

    Risk: Specs become stale
    Mitigation: Automated consistency checks in CI

    Risk: Tool initialization overhead
    Mitigation: Lazy-load specs only when needed

    Risk: Specs too verbose
    Mitigation: Use tiered detail (summary → full)

    Risk: Inconsistency with code
    Mitigation: Link specs to source code line numbers
