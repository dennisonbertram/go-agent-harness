Write content to a workspace file. Creates the file and any parent directories if they do not exist. Use this tool to create NEW files or to completely replace the contents of an existing file. If you only need to change part of a file (change a line, insert text, delete a section), prefer the edit tool instead — write replaces the entire file. Set "append" to true to add content to the end of an existing file without replacing it. The "expected_version" parameter enables optimistic concurrency: pass the version string from a previous read or write result, and the write will fail with a stale_write error if the file has been modified since then. Content can be provided via the "content" parameter (or its aliases: "new_text", "new_string", "text"). Path must be relative to the workspace root.

For files with a .json extension, the content is validated as JSON before being written. If the content is not valid JSON, the tool returns an error with code "invalid_json" and does not write the file. This prevents corrupting machine-readable files with malformed output. Fix the JSON and retry.
